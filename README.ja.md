# 🌸 Spring Boot RBAC 権限管理システム

Spring Boot と Spring Data JPA を使用して構築した **RBAC（Role-Based
Access Control）権限管理バックエンドシステム**です。

このプロジェクトでは、企業でよく使われる
**ユーザー・ロール・権限管理の基本構造**を実装しています。

## 📦 プロジェクト構成

    com.rbac
    ├── controller        # REST API コントローラー
    ├── service           # Service インターフェース
    ├── service.impl      # Service 実装
    ├── repository        # JPA リポジトリ
    ├── entity            # JPA エンティティ
    ├── dto               # リクエスト DTO
    ├── exception         # 例外処理
    └── common            # 共通レスポンスクラス

------------------------------------------------------------------------

## 🔐 RBAC モデル

本プロジェクトは標準的な **RBAC（Role-Based Access
Control）モデル**を採用しています。

    User
    Role
    Permission

    UserRole
    RolePermission

リレーション:

👤 **User ↔ Role** (多対多)\
🛡 **Role ↔ Permission** (多対多)

------------------------------------------------------------------------

## ✨ 主な機能

### 👤 ユーザー管理

-   ユーザー作成
-   ユーザー更新
-   ユーザー削除（論理削除）
-   ユーザー一覧取得
-   ユーザーへのロール割り当て

### 🛡 ロール管理

-   ロール作成
-   ロール更新
-   ロール削除
-   ロール一覧取得
-   ロールへの権限割り当て

### 🔑 権限管理

-   権限作成
-   権限更新
-   権限削除
-   権限一覧取得

------------------------------------------------------------------------

## 🚀 API 例

### ユーザー作成

POST /user/create

``` json
{
  "username": "admin",
  "password": "123456",
  "enabled": 1
}
```

### ユーザーにロールを割り当て

POST /user/{id}/roles

``` json
{
  "roleIds": [1,2]
}
```

### ロールに権限を割り当て

POST /role/{id}/permissions

``` json
{
  "permissionIds": [1,2,3]
}
```

## 🗄 データベース設計

主要テーブル:

    sys_user
    sys_role
    sys_permission
    sys_user_role
    sys_role_permission

本プロジェクトでは **論理削除（deleted フィールド）** を使用しています。

------------------------------------------------------------------------

## ⚙ 例外処理

以下のクラスを使用して **グローバル例外処理** を実装しています。

    BusinessException
    ErrorCode
    GlobalExceptionHandler

すべての API は **統一レスポンス形式** を返します。
